## 1. Обзор системы

Система предназначена для централизованного сбора, хранения и анализа информации об установленных программных пакетах на серверах и рабочих станциях. Основная задача — выявление уязвимостей (CVE) в установленном ПО и управление процессом их устранения.

**Архитектура:**
- Сервер на FastAPI + SQLite, работает в Docker-контейнере
- Агенты на целевых хостах отправляют список пакетов по HTTP
- SSH-сканер собирает пакеты без установки агента
- База уязвимостей NVD (National Vulnerability Database) хранится локально

---

## 2. Основные функции

| Функция | Описание |
|---------|----------|
| Сбор пакетов | Получение списка ПО с хостов через агент или SSH |
| CVE-сканирование | Проверка каждого пакета по локальной базе NVD |
| Управление статусами | Отслеживание жизненного цикла уязвимости: новая → в работе → исправлена |
| SLA-контроль | Автоматический дедлайн по критичности (Critical: 7 дней, High: 30, Medium: 90, Low: 180) |
| Риск-оценка | Расчёт приоритета по CVSS + EPSS + признаку эксплуатации (KEV) |
| Исключения (False Positive) | Пометка ложных срабатываний с указанием причины |
| Аудит | История изменения статусов по каждому пакету |
| Теги хостов | Классификация хостов (prod, dev, dmz и др.) |
| SSH-сканирование | Сбор пакетов напрямую по SSH без агента |
| Экспорт | Выгрузка отчётов в PDF и CSV |
| Тренды | График изменения количества уязвимостей по дням |

## 3. Вход в систему

Откройте браузер и перейдите по адресу сервера, например:

```
http://192.168.1.100:8000
```

Вы будете перенаправлены на страницу входа. Введите логин и пароль администратора.

После успешного входа открывается главный дашборд. Сессия хранится в cookie браузера.

Для выхода нажмите кнопку **Выйти** в правом верхнем углу или перейдите по адресу `/logout`.

## 4. Дашборд

**Адрес:** `/dashboard`

Дашборд — главная страница с общей статистикой по всем хостам.

### Статистические карточки

- **Всего пакетов** — общее число уникальных пакетов во всех отчётах
- **Уязвимых пакетов** — пакеты, для которых найдены CVE
- **Критических CVE** — уязвимости с CVSS ≥ 9.0
- **Высокий риск** — уязвимости с CVSS ≥ 7.0
- **⚠️ Просрочено SLA** — задачи, дедлайн которых уже прошёл
- **В работе** — пакеты со статусом «в задаче»

### График трендов

Линейный график показывает изменение за последние 30 дней:
- Синяя линия — общее число уязвимых пакетов
- Красная линия — критические уязвимости
- Оранжевая линия — пакеты с известным эксплоитом (⚡)

**Кнопка «Сделать снимок»** — вручную зафиксировать текущее состояние для графика. Снимки также делаются автоматически раз в сутки при старте сервера.

## 5. Хосты

**Адрес:** `/hosts`

Страница отображает все хосты, от которых получены отчёты.

### Таблица хостов

Столбцы:
- **Хост** — имя или IP-адрес
- **Теги** — метки классификации (prod, dev, dmz и др.)
- **Критичность** — числовое значение важности хоста (1–5)
- **Пакетов** — количество установленных пакетов
- **Уязвимостей** — число уязвимых пакетов
- **Последний отчёт** — дата получения последних данных
- **Статус** — онлайн/офлайн (проверяется по TCP-соединению)

### Управление тегами

1. Нажмите на имя тега в строке хоста или на кнопку редактирования
2. Введите теги через запятую, например: `prod, web, dmz`
3. Нажмите **Сохранить**

Доступные стандартные теги отображаются цветными метками: `prod` (красный), `dev` (синий), `dmz` (оранжевый), `test` (зелёный).

### Управление критичностью

Нажмите на значение критичности в строке хоста и выберите новое значение (1–5). Критичность влияет на расчёт итогового риска для хоста.

### SSH-сканирование хоста

Если для хоста настроены SSH-credentials (см. раздел 8), в строке появится кнопка **SSH Scan**. Нажмите её для запуска сканирования.

## 6. Пакеты

**Адрес:** `/packages`

Страница отображает все пакеты, найденные на хостах, с информацией о CVE.

### Фильтры

- По хосту — выберите конкретный хост из списка
- По имени пакета — поиск по названию
- Только уязвимые — скрыть пакеты без CVE

### Информация по пакету

Нажмите на строку пакета для просмотра деталей:
- Список CVE с идентификаторами, описанием и CVSS-оценкой
- Метка ⚡ — уязвимость активно эксплуатируется (в базе KEV или EPSS ≥ 0.7)
- Метка 🔴/🟠/🟡 — цветовая маркировка по критичности

### Перезапуск CVE-проверки

Нажмите **«Пересканировать»** в строке пакета, чтобы заново проверить его по базе NVD. Удобно после обновления базы.

## 7. Управление уязвимостями

**Адрес:** `/software-management`

Это основная рабочая страница для аналитика безопасности. Здесь управляется жизненный цикл каждой уязвимости.

### Вкладки фильтрации

- **Все** — полный список пакетов
- **Новые** — ещё не обработанные
- **В задаче** — взяты в работу, есть дедлайн
- **Игнорировать** — помечены как неактуальные
- **Исправлено** — уязвимость устранена

### Таблица пакетов

Столбцы:
- **Название / Версия** — имя пакета и версия
- **CVE** — количество найденных уязвимостей
- **CVSS** — максимальная оценка серьёзности (0–10)
- **EPSS** — вероятность эксплуатации (0–1)
- **KEV** — признак активной эксплуатации (⚡)
- **SLA** — дедлайн для устранения (красный = просрочено)
- **Статус** — текущее состояние

### Изменение статуса пакета

1. Нажмите на строку пакета — откроется панель редактирования
2. Выберите статус:
   - **Новый** — уязвимость обнаружена, не обработана
   - **В задаче** — создана задача на устранение (автоматически устанавливается дедлайн SLA)
   - **Игнорировать** — уязвимость неактуальна для данной среды
   - **Исправлено** — пакет обновлён или удалён
3. Добавьте комментарий при необходимости
4. Нажмите **Сохранить**

### SLA-дедлайны

При установке статуса «В задаче» автоматически рассчитывается дедлайн:

| Критичность CVSS | Срок |
|-----------------|------|
| Критическая (≥ 9.0) | 7 дней |
| Высокая (≥ 7.0) | 30 дней |
| Средняя (≥ 4.0) | 90 дней |
| Низкая (< 4.0) | 180 дней |

Просроченные задачи подсвечиваются красным в таблице и учитываются в счётчике на дашборде.

### Массовое обновление

1. Отметьте несколько пакетов галочками
2. Нажмите кнопку **Массовое обновление**
3. Выберите новый статус и при необходимости комментарий
4. Подтвердите действие

### История статусов

В панели редактирования пакета внизу отображается история всех изменений: кто (статус), когда и с каким комментарием.

### Исключения (False Positive)

Если конкретная CVE не применима к данному пакету (например, уязвимость только для Windows, а у вас Linux):

1. Откройте список CVE для пакета
2. Рядом с нужной CVE нажмите **FP** (False Positive)
3. Введите причину исключения
4. CVE будет скрыта из результатов для этого пакета

Для просмотра и управления всеми исключениями используйте API: `GET /api/vuln-exceptions`.

### Перепроверка после исправления

Для пакетов со статусом **«В задаче»** доступна кнопка **«🔁 Перепроверить»**. После нажатия система заново проверяет пакет по NVD. Если CVE больше не найдены — статус автоматически меняется на **«Исправлено»**.

### Нормализация имени для NVD

Каждый пакет имеет поле «Имя для NVD» — это имя, которое используется при поиске в базе уязвимостей. Если автоматически подобранное имя не даёт результатов:
1. Нажмите на пакет
2. Отредактируйте поле «Имя для NVD»
3. Нажмите **Сохранить** — система сразу пересканирует пакет с новым именем

## 8. SSH-сканирование

SSH-сканирование позволяет собирать список установленных пакетов с хостов **без установки агента**. Система подключается по SSH и выполняет команды `dpkg-query`, `rpm -qa` или `apk info` в зависимости от ОС.

### Поддерживаемые системы

| ОС | Команда |
|----|---------|
| Debian, Ubuntu, Kali | `dpkg-query` |
| RHEL, CentOS, Fedora, Rocky | `rpm -qa` |
| Alpine Linux | `apk info` |

### Шаг 1. Добавить SSH-credentials

```
POST /api/ssh-credentials
Content-Type: application/json

{
  "hostname": "192.168.1.50",
  "port": 22,
  "username": "admin",
  "auth_type": "password",
  "secret": "пароль",
  "use_sudo": false
}
```

Для аутентификации по ключу:

```
POST /api/ssh-credentials
Content-Type: application/json

{
  "hostname": "192.168.1.50",
  "port": 22,
  "username": "deploy",
  "auth_type": "key",
  "key_path": "/data/ssh_keys/id_rsa",
  "secret": "passphrase_если_есть",
  "use_sudo": false
}
```

Пароль хранится в зашифрованном виде (Fernet AES-128). Из API пароль никогда не возвращается.

### Шаг 2. Запустить сканирование

```
POST /api/ssh-scan/192.168.1.50
```

Ответ:
```json
{
  "ok": true,
  "scan_id": 42,
  "hostname": "192.168.1.50"
}
```

Сканирование выполняется в фоне. Повторный запрос вернёт ошибку 409 если scan уже запущен.

### Шаг 3. Проверить статус

```
GET /api/ssh-scan/status/192.168.1.50
```

Ответ:
```json
{
  "status": "completed",
  "total_packages": 487,
  "started_at": "2026-04-26T13:00:00",
  "completed_at": "2026-04-26T13:00:15"
}
```

Возможные статусы: `never`, `pending`, `processing`, `completed`, `failed`.

### Шаг 4. Просмотреть результаты

После завершения сканирования хост и его пакеты появятся на страницах **Хосты** и **Пакеты**. Автоматически запускается CVE-проверка по всем найденным пакетам.

### Безопасность SSH

- Используйте отдельного пользователя с минимальными правами (только чтение списка пакетов)
- `dpkg-query` и `rpm -qa` не требуют root
- `use_sudo: true` включайте только если команды требуют привилегий
- Приватные ключи храните в директории `/data/ssh_keys/` (вне Docker-образа, в volume)
- Установите переменную окружения `SSH_SECRET_KEY` для шифрования паролей:
  ```bash
  SSH_SECRET_KEY=ваш_случайный_ключ_32_символа
  ```

## 9. Экспорт отчётов

### CSV-экспорт

**Адрес:** Кнопка **«Экспорт CSV»** на странице управления уязвимостями, или напрямую:

```
GET /api/export/csv?filter=all
```

Параметр `filter`: `all`, `new`, `in_task`, `ignore`, `fixed`

Файл открывается в Excel с корректным отображением кириллицы (кодировка UTF-8 с BOM).

**Столбцы в файле:**
Название, Версия, Хосты, Имя для NVD, Статус, CVE, CVSS макс., EPSS, KEV, Последняя проверка, Дедлайн SLA, Просрочено, Комментарий

### PDF-экспорт

**Адрес:** Кнопка **«Экспорт PDF»** на странице управления уязвимостями, или напрямую:

```
GET /api/export/pdf
```

Отчёт содержит:
- Заголовок с датой генерации
- Сводную статистику (всего пакетов, уязвимых, критических, в работе, просрочено)
- Таблицу уязвимых пакетов с CVE, CVSS, EPSS, статусом и дедлайном

**Требование:** В директории `/app/fonts/` должен быть файл `DejaVuSans.ttf` для поддержки кириллицы. Если шрифт отсутствует — текст будет в латинице.

## 10. Сбор данных с агента

Агент устанавливается на целевой хост и периодически отправляет список пакетов на сервер.

### Формат данных

Агент отправляет POST-запрос:

```
POST /api/collect
Content-Type: application/json
X-API-KEY: ваш_ключ  (если настроен API_KEY)
```

Тело запроса:
```json
{
  "hostname": "server-01",
  "ip": "192.168.1.10",
  "os": "Ubuntu 22.04",
  "collected_at": "2026-04-26T10:00:00Z",
  "software": [
    {"name": "openssl", "version": "3.0.2"},
    {"name": "nginx", "version": "1.24.0"},
    {"name": "python3", "version": "3.10.12"}
  ]
}
```

### Пример запуска агента вручную (Linux)

```bash
dpkg-query -W -f='${Package}\t${Version}\n' | python3 -c "
import sys, json, socket, requests
from datetime import datetime, timezone

packages = []
for line in sys.stdin:
    parts = line.strip().split('\t')
    if len(parts) == 2:
        packages.append({'name': parts[0], 'version': parts[1]})

payload = {
    'hostname': socket.gethostname(),
    'ip': '',
    'os': 'Linux/Debian',
    'collected_at': datetime.now(timezone.utc).isoformat(),
    'software': packages
}

r = requests.post('http://192.168.1.100:8000/api/collect', json=payload)
print(r.json())
"
```

### Переменные окружения сервера

| Переменная | Описание | По умолчанию |
|------------|----------|--------------|
| `API_KEY` | Ключ для защиты API сбора данных | не задан (открытый) |
| `NVD_API_KEY` | Ключ NVD API для повышенного лимита запросов | не задан |
| `SSH_SECRET_KEY` | Ключ шифрования SSH-паролей | встроенный (небезопасно) |
| `DB_PATH` | Путь к файлу SQLite | `/data/reports/vuln_collector.db` |
| `DATA_DIR` | Директория для JSON-отчётов | `/data/reports` |

## Часто задаваемые вопросы

**Q: Пакет найден, но CVE не показываются.**
A: Возможно, имя пакета не совпадает с именем в NVD. Откройте пакет в управлении уязвимостями и скорректируйте поле «Имя для NVD».

**Q: SSH-сканирование завершается с ошибкой «Authentication failed».**
A: Проверьте логин и пароль. Убедитесь что пользователь может зайти по SSH вручную: `ssh user@host`.

**Q: График трендов пустой.**
A: Нажмите кнопку «Сделать снимок» на дашборде. Данные появляются только после первого снимка.

**Q: PDF-отчёт генерируется, но текст нечитаемый (вопросики вместо кириллицы).**
A: Скопируйте файл `DejaVuSans.ttf` в директорию `server/fonts/` на хосте. Перезапуск контейнера не требуется.

**Q: Статус хоста «офлайн», хотя хост доступен.**
A: Проверка статуса выполняется по TCP-порту 22 (SSH). Если SSH закрыт или порт изменён — хост будет отображаться как офлайн.
